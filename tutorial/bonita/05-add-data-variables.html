Exercice 2 (c): Define data variables

This exercice is define the data variables need to used during the execution phase.

Extract the data variable need to use in the implementation phase

Based on the process description, you should be abel to extract the data variables need to use during the execution phase. There are two data types:
  1. Business data: there are the data need to be stored within the database (BDM).
  2. Process variable: there are the data need to used during the execution of each process. These variables are used temporary during the execution. For example for get data from user and the do some mathematical function before save in the database. Also, can be used to add the condition on the gateways.

How to add Business data and process variable?

  1. To add data variables, you need to select the pool, and then within "Data" tab, select "pool variables", and then you can add Business variables (for BDM objects), and Process variables (used during the execution).

  2. To add a business data, after click on add in "Business variable", you can add name and the the Business Object (from the BDM object).

  3. To add a business variable, after click on add in "Process variable", you can add name and the the data type (Text, double, ...).



Extract the mission data variable from the process description

In the process description, you can find in bold the data variables need to be used during the execution phase. Also, in italic the attributes that used late in the implementation. However, there still missing one Business data. Please find it and add to to you process data.

A user fills a procurement request (summary and description) and identify a supplier. This request is sent to the supplier a quotation. The supplier should add the following information to the quotation: price, comments, if he is accept or no. The status of quotation will automatically changed to completed. After completed, the quotations are sent back to the requestor for review and selection. Before starting the review process, the status of the request will be changed to "pending for review". In the review quotation, the requestor should accept or reject the quotation. And finally based on the decision of the requestor, the request status changed to completed or abort. To effectively implement the process, developer must understand how the information is conceptually structured and arranged within each class (suppliers, quotation and request) and how the process interacts with it.

Add the missing data variable to your process model.
